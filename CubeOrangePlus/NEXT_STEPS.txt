================================================================================
                          HARDWARE TO GET
================================================================================

BATTERY
-------
Option 1: 2S-3S LiPo (7.4V-11.1V)
  - 2200-5000mAh capacity
  - XT60 connector (standard)
  - Need: LiPo charger, voltage alarm

Option 2: 2S-3S 18650 Li-ion pack
  - Longer runtime, safer chemistry
  - Need: BMS board, charger

Recommended: 3S 2200mAh LiPo for good power/weight balance

RC TRANSMITTER + RECEIVER
-------------------------
For manual override and mode switching:
  - FlySky FS-i6X (~$50) - good budget option
  - RadioMaster Zorro (~$100) - better quality
  - Receiver connects to Cube RCIN port

Minimum 6 channels:
  - CH1: Steering
  - CH2: Throttle
  - CH3: Mode switch
  - CH4-6: Aux functions

GPS MODULE (if not included with Cube carrier)
----------------------------------------------
  - Here3 or Here4 GPS (CAN bus, plug and play)
  - Or generic UBLOX M8N/M10 (UART)
  - Required for: AUTO mode, RTL, waypoint navigation

TELEMETRY (optional)
--------------------
  - SiK Radio pair (433MHz or 915MHz)
  - Connects to Cube TELEM1 port
  - Allows wireless Mission Planner connection

COMPONENTS TO HOUSE
-------------------
  - Cube Orange Plus (40x40mm mounting)
  - ESP32 WROOM
  - L298N motor driver
  - GPS module + mast
  - Battery compartment
  - RC receiver

DESIGN CONSIDERATIONS
---------------------
  - Ventilation for L298N (gets hot)
  - USB access for ESP32 programming
  - Easy battery swap
  - GPS mounted high, away from motors
  - Cube orientation marked (arrow forward)
  - Wire routing channels

MOUNTING
--------
  - Cube: M3 screws, 40mm square pattern
  - ESP32: M2.5 screws or friction fit
  - L298N: M3 screws, ~43x43mm pattern
  - Consider vibration dampening for Cube

================================================================================
                     CAPABILITIES WITH CUBE ORANGE PLUS
================================================================================

AUTONOMOUS NAVIGATION
---------------------
  - Waypoint missions (plan in Mission Planner, execute in AUTO mode)
  - Return to Launch (RTL) - automatic return home
  - Loiter - hold position at GPS coordinate
  - GeoFence - stay within defined boundary

FOLLOW MODE
-----------
  - Follow a moving target (phone GPS, another vehicle)
  - Requires telemetry link

GUIDED MODE
-----------
  - Send real-time position commands via MAVLink
  - Control from custom app or companion computer

OBJECT AVOIDANCE (future)
-------------------------
  - Add ultrasonic sensors or lidar
  - Cube supports proximity sensors via I2C or CAN

COMPANION COMPUTER (future)
---------------------------
  - Add Raspberry Pi for computer vision
  - Camera-based lane following
  - Object detection and avoidance
  - Connect via Cube TELEM2 port

DATA LOGGING
------------
  - Onboard SD card logs all sensor data
  - Review with Mission Planner log analyzer
  - GPS tracks, speed, heading, battery, etc.

================================================================================
                          TESTING PROGRESSION
================================================================================

1. BATTERY TEST (indoor)
   - Connect battery instead of power supply
   - Verify all voltages stable
   - Test full throttle current draw
   - Check for brownouts

2. MANUAL DRIVE TEST (outdoor, open area)
   - RC transmitter control
   - Test steering range and response
   - Test speed control
   - Practice emergency stop

3. HOLD MODE TEST
   - Switch to HOLD mode while moving
   - Verify vehicle stops and holds

4. AUTO MODE TEST (simple mission)
   - Create 3-4 waypoint square
   - Low speed (1-2 m/s)
   - Open area with good GPS
   - Be ready to switch to MANUAL

5. ADVANCED TESTING
   - Longer missions
   - RTL from various positions
   - GeoFence breach response
   - Failsafe testing
